Project Overview

What is PostAI?

PostAI is an intelligent logistics optimization platform that solves the Traveling Salesman Problem (TSP) for delivery route planning. It uses advanced algorithms to find the most efficient route for visiting multiple delivery locations.

The system features a hybrid approach that guarantees optimal solutions for small networks (≤12 locations) and near-optimal solutions for larger networks, with priority handling for urgent deliveries.

Graph (Adjacency List)

LogisticsGraph Class

Space: O(V + E) Insert: O(1)

What is it?

A Graph represents the network of delivery locations. Each location is a vertex and the roads between them are edges with distances as weights.

Why Adjacency List?

  • check_circle Memory Efficient: O(V+E) vs O(V²) for matrix
  • check_circle Faster Iteration: Only visits actual neighbors
  • check_circle Sparse Networks: Real-world roads are sparse 
// Graph Data Structure (Adjacency List)
class LogisticsGraph {
private:
    int numVertices;
    vector<vector<Edge>> adjacencyList;
    unordered_map<int, Location> locations;
    
public:
    LogisticsGraph(int vertices) : 
        numVertices(vertices) {
        adjacencyList.resize(vertices);
    }
    
    void addEdge(int src, int dest, double dist) {
        adjacencyList[src].push_back(
            Edge(dest, dist)
        );
    }
    
    const vector<Edge>& getNeighbors(int id) {
        return adjacencyList[id];
    }
};

Stack

Undo History Feature

Push: O(1) Pop: O(1)

What is it?

A Stack is a LIFO (Last-In-First-Out) data structure. Like a stack of plates - you can only add or remove from the top.

How it's used?

Every time you add locations, the current state is pushed onto the stack. When you click Undo, the last state is popped and restored. Perfect for undo functionality!

lightbulb

LIFO behavior perfectly matches how undo should work - restore the most recent action first!

// Stack for Undo History
stack<vector<Location>> historyStack;

// PUSH - Save state before changes
historyStack.push(locations);
cout << "State saved (Total: " 
     << historyStack.size() << ")";

// POP - Restore previous state
if (!historyStack.empty()) {
    vector<Location> previous = 
        historyStack.top();
    historyStack.pop();
    // Restore the previous state
}

Priority Queue (Min-Heap)

Prim's MST Algorithm

Insert: O(log n) Extract: O(log n)

What is it?

A Priority Queue always gives you the element with the smallest (or largest) priority first. Implemented as a binary heap.

How it's used?

In Prim's Algorithm for building the Minimum Spanning Tree (MST), we always need the shortest edge next. The priority queue efficiently maintains edges sorted by weight.

  • arrow_right Always extracts the minimum cost edge
  • arrow_right Maintains heap property automatically 
// Priority Queue Node
struct PriorityNode {
    int node;
    double cost;
    int parent;
    
    bool operator>(const PriorityNode& other) {
        return cost > other.cost;
    }
};

// Min-Heap Priority Queue
priority_queue<
    PriorityNode,
    vector<PriorityNode>,
    greater<PriorityNode>  // Min-heap
> pq;

pq.push(PriorityNode(0, 0.0, -1));

while (!pq.empty()) {
    PriorityNode curr = pq.top();
    pq.pop();  // Always gets minimum
}

2D Vector (Distance Matrix)

Dynamic Programming Table

Space: O(n²) Access: O(1)

What is it?

A 2D Vector (matrix) stores distances between all pairs of locations. Cell [i][j] contains the distance from location i to location j.

Why use it?

The Held-Karp DP algorithm requires frequent distance lookups. Pre-computing all distances gives O(1) access time during optimization. 

// Distance Matrix for DP
vector<vector<double>> distMatrix;

void buildDistanceMatrix() {
    int n = allLocations.size();
    distMatrix.resize(n, vector<double>(n));
    
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            if (i != j) {
                distMatrix[i][j] = 
                    haversineDistance(
                        locations[i].lat,
                        locations[i].lon,
                        locations[j].lat,
                        locations[j].lon
                    );
            }
        }
    }
}

Bitmask (Integer as Set)

Held-Karp State Representation

Space: O(1) Set Ops: O(1)

What is it?

A Bitmask uses an integer's binary representation to store a set of elements. Each bit represents whether an element is included (1) or not (0).

Why use it?

In Held-Karp DP, we need to track which cities have been visited. Instead of using arrays, a single integer can represent all 2n possible subsets!

Example: mask = 13 = 1101₂ means cities {0, 2, 3} are visited

// Bitmask Operations in Held-Karp
int fullMask = (1 << n) - 1; // All visited

// Check if city i is in mask
if (mask & (1 << i)) { // Visited }

// Add city next to mask
int newMask = mask | (1 << next);

// Remove city curr from mask
mask ^= (1 << curr);

// DP state: dp[mask][i]
// = min cost to visit all cities in mask,
//   ending at city i
